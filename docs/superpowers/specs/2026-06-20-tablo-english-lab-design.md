# Tablo English Lab — 설계 문서

- **작성일**: 2026-06-20
- **목적**: YouTube 영어 영상(예: HEYTABLO 팟캐스트)으로 듣기·표현·쉐도잉·복습을 순환 학습하는 개인용 웹앱
- **대상 기기**: Samsung S24 (모바일 우선), 개인 학습용 배포
- **상태**: 설계 승인 완료 → 구현 계획 작성 단계

---

## 1. 개요 & 목표

기존 단일 HTML 학습 노트(`english_study_notebook.html`)를 확장하여, 4개 핵심 모듈과 SRS 순환 구조를 가진 모바일 웹앱(PWA)으로 만든다.

핵심 학습 루프:

```
영상+자막 (입력) ──┬──> 표현 추출/단어장 (분기 A) ──┐
                   └──> 쉐도잉 녹음 비교 (분기 B) ──┤
                                                     ├──> 퀴즈/복습 SRS (합류)
                                  ┌──── 점선 루프 ───┘
                                  └─> 틀린 항목 다시 단어장으로 순환
```

설계 원칙:
- **영어는 권장이지만 선택**: 영어 자막이 없어도 한국어만으로 등록·재생·A-B 루프가 동작하고, 귀로 들으며 영어 표현을 직접 채우는 것 자체가 학습이 된다.
- **모바일 우선**: S24 세로 화면 기준 레이아웃.
- **비용 0원**: 백엔드·로그인·유료 서비스 없음.
- **오프라인 동작**: PWA + IndexedDB.

## 2. 기술 스택 & 배포

| 항목 | 선택 |
|---|---|
| 프레임워크 | React + Vite |
| 앱 형태 | PWA (홈 화면 추가, 오프라인, `navigator.storage.persist()`) |
| 저장소 | IndexedDB (Dexie 래퍼) |
| 백업 | JSON 내보내기 / 가져오기 (수동) |
| 영상 | YouTube IFrame Player API (`enablejsapi=1`) |
| 녹음 | MediaRecorder API + Web Audio (파형) |
| 배포 | Vercel 무료 티어 |
| 디자인 톤 | 기존 종이 노트 감성 (앰버+틸, Fraunces/Inter) 유지·발전 |

**비용**: 월 0원 / 연 0원. 과금 지점 없음.

## 3. 데이터 모델 (IndexedDB / Dexie)

```
videos      { id, youtubeId, title, createdAt, lastStudiedAt }
chapters    { id, videoId, title, startTime, order }
lines       { id, videoId, chapterId, time(sec), en, ko, order }
vocab       { id, videoId, lineId, term, note(ko 기본값), tags[],
              type: 'word'|'idiom'|'slang',
              srs: { box, dueAt, history[] }, createdAt }
recordings  { id, videoId, lineId|segment, audioBlob, takeNo, createdAt }
settings    { key, value }   // 영구저장 동의, 토글 상태 등
```

- 백업: 위 전체를 하나의 JSON으로 내보내기/가져오기 (오디오 blob은 base64 또는 별도 처리).

## 4. 모듈 1 — 영상 + 자막 (입력/학습 허브)

### 4.1 영상 라이브러리
- **여러 영상 = 섹션**으로 구분. 라이브러리 화면에서 영상 카드 목록.
- 영상 추가: YouTube ID/URL 입력 → 제목 지정.

### 4.2 영상 재생 & A-B 루프 (핵심)
- YouTube **IFrame Player API**로 정밀 제어.
- **A-B 루프**: 구간(A~B) 지정 후 무한 반복. `currentTime` 폴링으로 B 도달 시 A로 seek.
- 재생 속도 조절: 1x / 0.75x / 0.5x.
- 반복 횟수 표시.

### 4.3 자막 입력 & 파서
지원 입력:
1. **수동 붙여넣기** (YouTube '스크립트 표시' 복사)
2. **`.srt` / `.vtt` 파일 업로드** (타임코드 정밀)
3. 영어 / 한국어 **각각** 입력 가능 (둘 다, 또는 한쪽만)

영어 자막이 없을 때: downsub / notegpt / tactiq 등 무료 Whisper 도구로 `.srt` 생성 → 업로드. ASR 오류는 앱 내 인라인 수정으로 교정.

파서 자동 처리:
- **`챕터 N: 제목`** 형식 인식 → 챕터로 그룹화 (접기/펼치기).
- **중복 제거**: `0:11` 뒤에 붙는 한국어 시간 라벨(`11초`, `1분 5초`) 자동 제거. 반복 문장 정리.
- 다양한 타임코드 인식: `0:05`, `00:05`, `1:02:03`.
- 영어·한국어 두 트랙을 **시간 기준 정렬**.

### 4.4 자막 보기
- **영어 크게 / 한국어는 토글로 숨김·표시** (기본: 한국어 숨김).
- 줄 옆 ▶: 그 구간 재생. 줄 길게 누르기: A-B 루프 구간 설정.
- **자막 줄 인라인 수정** (ASR 오류 교정 / 받아쓰기 연습).

---

## 5. 모듈 2 — 표현 추출 / 단어장 (분기 A)

- 영어 표현 **클릭/드래그** → 단어장 추가.
- **메모 기본값 = 그 줄의 한국어 번역** 자동 채움 (예: `People Scare Me` → "난 사람이 제일 무서워").
- 태그 분류: **단어 / 관용구 / 슬랭** (Tablo 구어체 대응). 예: `Fake it till you Make it` → 슬랭.
- 카드에서 **원본 영상·시간으로 점프**.
- 카드 메모 수정·삭제.

---

## 6. 모듈 3 — 쉐도잉 녹음 비교 (분기 B)

- 줄/구간 선택 → 원본 재생 → **MediaRecorder로 내 녹음**.
- **원본 ↔ 내 녹음 번갈아 빠르게 재생**해 억양·속도 비교.
- 내 녹음 **파형(Web Audio) 표시**, 여러 테이크 저장.
- 0.5x 느리게 비교.

### 한계 (명시)
YouTube 오디오는 교차출처(cross-origin)라 **원본 파형을 나란히 겹쳐 분석하지 못한다.** 따라서 비교는 "원본 듣기 ↔ 내 녹음 듣기" 토글 + 느린 재생 + 내 녹음 파형 표시로 구현한다. (원본/내 녹음 파형 오버레이는 범위 외.)

---

## 7. 모듈 4 — 퀴즈 / 복습 (SRS, 합류점)

- 단어장 표현으로 퀴즈 자동 생성:
  - **빈칸 채우기**: 영어 문장에서 표현을 가리고 맞히기 (자막 문맥 사용).
  - **듣기 퀴즈**: A-B 구간 듣고 표현 맞히기.
- **SRS (Leitner 박스 방식)**: 맞히면 복습 간격↑(다음 박스), 틀리면 박스 강등 → 곧 다시 등장.
- **오늘 복습할 카드 수** 배지 표시.
- 점선 루프: 틀린 항목이 다시 단어장 복습 큐로 순환.

## 8. 디자인 / UX

- 모바일 우선 (S24 세로). 하단 탭 또는 단일 스크롤 + 모듈 전환.
- 기존 HTML의 종이 노트 감성 유지: `--paper`, `--amber`, `--teal`, Fraunces(제목)/Inter(본문).
- 핵심 인터랙션: 자막 줄 탭 → 재생 / 길게 누르기 → 루프 / 단어 탭 → 단어장.

## 9. 범위 (Scope)

### 포함
- 영상 라이브러리(여러 영상=섹션), 챕터 그룹화, 중복 제거 파서
- 이중 자막(영-한) + 한국어 토글
- A-B 루프, 속도 조절
- 단어장(태그, 한국어 자동 메모, 점프)
- 쉐도잉 녹음(번갈아 재생, 내 녹음 파형)
- 퀴즈(빈칸/듣기) + SRS
- IndexedDB 저장 + JSON 백업/복원 + PWA

### 제외 (YAGNI / 범위 외)
- 백엔드·로그인·클라우드 동기화
- 자동 자막 가져오기(서버리스 함수)
- 원본↔내녹음 파형 오버레이 분석
- 다중 사용자
